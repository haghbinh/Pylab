    پروژه پایانی پایتون متوسطه: مدیریت فروشگاه آنلاین کوچک

    در این پروژه شما یک برنامه مدیریت فروشگاه آنلاین کوچک خواهید ساخت که قابلیت مدیریت محصولات، موجودی و سفارش‌ها را دارد.
    هدف از این پروژه تمرین استفاده از مفاهیم پایتون متوسطه شامل توابع پیشرفته، Lambda، map، filter، zip، Decorators، Closure، ماژول‌ها و مدیریت استثناها است.

    شرح پروژه

    برنامه باید قابلیت‌های زیر را داشته باشد:

    1. اضافه کردن محصول جدید با مشخصات (نام محصول، قیمت، تعداد موجودی).
    2. مشاهده لیست محصولات و وضعیت موجودی هر محصول.
    3. ثبت سفارش برای یک یا چند محصول و کاهش موجودی آن‌ها.
    4. بازگرداندن محصول و افزایش موجودی.
    5. استفاده از Decorator برای ثبت تراکنش‌ها (Log) و نمایش پیغام قبل و بعد عملیات.
    6. استفاده از Closure برای نگهداری تعداد دفعات سفارش هر محصول.
    7. استفاده از Lambda، map و filter برای جستجو و فیلتر کردن محصولات.
    8. مدیریت خطاها شامل سفارش محصول ناموجود، وارد کردن تعداد منفی و ورودی‌های نامعتبر.
    9. امکان ذخیره‌سازی داده‌ها در فایل برای استفاده مجدد در اجرای بعدی برنامه.

    پیشنهاد ساختار فایل‌ها و ماژول‌ها

    می‌توانید پروژه را به چند فایل ماژول تقسیم کنید:

    • products.py : شامل کلاس Product و توابع مدیریت محصولات
    • orders.py : شامل کلاس Order و توابع مدیریت سفارش‌ها
    • transactions.py : شامل Decorator برای ثبت تراکنش‌ها و گزارش عملیات
    • main.py : برنامه اصلی که رابط کاربری متنی دارد و همه ماژول‌ها را وارد می‌کند

    نکات مهم در پیاده‌سازی

    • برای جستجو و فیلتر محصولات از filter و lambda استفاده کنید.
    • برای به‌روزرسانی موجودی محصولات از map و zip بهره ببرید.
    • Decoratorها را برای ثبت تراکنش‌ها و نمایش پیغام قبل و بعد از عملیات به کار ببرید.
    • Closureها را برای نگهداری تعداد دفعات سفارش هر محصول استفاده کنید.
    • مدیریت خطاها شامل سفارش محصول ناموجود، تعداد منفی یا ورودی نامعتبر باشد.
    • ذخیره و بازیابی داده‌ها با فایل باعث می‌شود فروشگاه دائماً قابل استفاده باشد.

    تمرین‌ها و چالش‌ها

    ۱. لیستی از محصولات بسازید و با Lambda و filter فقط محصولات موجود را نمایش دهید.
    ۲. یک Decorator بسازید که هر بار سفارشی ثبت شد تراکنش را در فایل لاگ ذخیره کند.
    ۳. Closure بسازید که تعداد سفارش هر محصول را نگه دارد و امکان گزارش گیری بدهد.
    ۴. هنگام وارد کردن سفارش‌ها، خطاهای احتمالی را با try-except مدیریت کنید.
    ۵. با استفاده از map و zip موجودی محصولات را بعد از سفارش یا بازگشت به‌روز کنید.
    ۶. رابط کاربری متنی بسازید که کاربر بتواند عملیات مختلف (افزودن محصول، سفارش، نمایش موجودی) را انتخاب کند.
    ۷. داده‌ها را در فایل ذخیره کنید و برنامه را طوری طراحی کنید که در اجرای بعدی بتواند اطلاعات قبلی را بارگذاری کند.
